# 协作流程

## 入职第一天

- 得到 Git 远程仓库的地址和账号密码

- 将代码克隆到本地（地址换成自己的）

  ```shell
  git clone https://github.com/XXXXXXX/test.git
  ```

- 切换分支

  ```
  git checkout -b XXXXXX
  ```

- 开发代码

- 本地提交

  ```shell
  git add -A
  git commit -m '注释内容'
  ```

- 合并分支

  ```shell
  git checkout master
  git merge XXXXXX
  ```

- 更新本地代码

  ```shell
  git pull
  //或者使用下面两行代码
  git fetch
  git merge origin/master
  ```

- 提交代码

  ```shell
  git push
  ```

## 第二天工作流程

1. 更新代码 (git pull)
2. 切换分支
3. 开发功能
4. 提交
5. 合并分支
6. 更新代码
7. 提交代码

#### 冲突解决

同分支冲突一样的处理，将代码调整成最终的样式，提交代码即可。
